Source stat block data from SteelCompendium bestiary — bestiary.ts loads data/bestiary/Monsters/statblocks.json (416 monsters), builds name-keyed lookup map, maps raw JSON → MonsterFeature[]

**Key decisions:**
- Direct JSON import of statblocks.json via Vite's resolveJsonModule; no runtime fetch needed
- baseName() strips trailing ordinal ("Goblin Assassin 1" → "Goblin Assassin") so numbered encounter copies resolve to the base statblock

- mapFeatures() filters to ability/trait only (excludes subtrait), omits optional fields when absent

[MINION-004]

Interval-based shared stamina pool for minion groups — replaces standard `cur/max` with threshold markers (e.g. `5 / 10 / 15 / 20`)

**Key decisions:**
- `minionInterval()` in bestiary.ts resolves per-minion stamina from bestiary; tries parent name, falls back to first minion name (same pattern as resolveFeatures)
- `minionThresholds()` builds ascending threshold array: `interval * (i+1)` for each minion
- MinionStaminaDisplay component shows threshold segments with 3 visual states: healthy (white), at-risk (amber, stamina within interval but below threshold), dead (red + line-through, stamina at or below previous threshold)
- EditableStaminaCell gets optional `renderDisplay` prop so MinionGroupRow can inject the interval display while keeping the existing hover popover for stamina adjustments
- Seed data updated: Minions stamina `[12,12]` → `[20,20]` to match 4×5 bestiary interval total
